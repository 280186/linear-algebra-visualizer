# 线性代数二维线性变换演示软件项目规格

---

## 1. 项目定位

本项目是一个二维线性代数交互式演示软件。

最终目标是接近 3Blue1Brown 的数学动画风格，但项目形式不是视频，而是可以在浏览器中实时交互的 Web 应用。

软件主要演示：

1. 矩阵表示线性变换；
2. 矩阵乘积表示线性变换组合；
3. 基变换；
4. 坐标变换；
5. 行列式的面积缩放意义；
6. 逆矩阵；
7. 特征向量；
8. 同一个向量在不同基下的坐标变化。

---

## 2. 技术栈

主技术栈：

```text
TypeScript + Vite + React + Canvas
```

后期可加入：

```
SVG + KaTeX + PixiJS/WebGL
```

当前阶段不要引入过多复杂依赖。

------

## 3. 核心设计思想

本项目不是一个简单的矩阵计算器，而是一个小型数学动画系统。

它最终应该具备以下能力：

1. 有数学对象，例如向量、矩阵、网格、单位方格；
2. 有场景对象，例如坐标轴、箭头、标签、公式；
3. 有相机系统，负责世界坐标和屏幕坐标转换；
4. 有动画系统，负责对象随时间变化；
5. 有教学模块，负责把抽象公式和图形动画对应起来。

------

## 4. 四阶段开发路线

# 第一阶段：Web 最小原型

## 目标

完成一个最小可用 Demo。

用户可以输入一个 2×2 矩阵 A，然后点击播放，看到二维平面从原始状态平滑变化到 A 作用后的状态。

## 第一阶段功能

必须实现：

1. 页面中显示 Canvas；
2. Canvas 中显示二维坐标轴；
3. Canvas 中显示网格；
4. Canvas 中显示标准基向量 e1、e2；
5. Canvas 中显示单位正方形；
6. 页面中有 2×2 矩阵输入框；
7. 页面中有“播放变换”按钮；
8. 页面中有“重置”按钮；
9. 点击播放后，网格、基向量、单位正方形平滑变换；
10. 非法矩阵输入不能导致程序崩溃。

## 第一阶段数学公式

动画插值矩阵：

```
M(t) = (1 - t)I + tA
```

其中：

```
0 <= t <= 1
```

每一帧使用当前矩阵 M(t) 作用到点：

```
v_t = M(t) v
```

## 第一阶段不要实现

暂时不要实现：

1. 矩阵乘积；
2. 基变换；
3. 坐标变换；
4. 特征值；
5. 特征向量；
6. 复杂动画系统；
7. WebGL；
8. 拖拽交互；
9. 导出视频。

# 第二阶段：抽象数学对象和场景对象

## 目标

把第一阶段的临时代码重构为长期可扩展的结构。

## 需要抽象的数学对象

### Vec2

二维向量对象。

建议方法：

```
add(v: Vec2): Vec2
sub(v: Vec2): Vec2
scale(k: number): Vec2
dot(v: Vec2): number
length(): number
normalized(): Vec2
clone(): Vec2
```

### Mat2

二维矩阵对象。

建议方法：

```
static identity(): Mat2
static rotation(theta: number): Mat2
static scale(sx: number, sy: number): Mat2
static shear(kx: number, ky: number): Mat2

multiplyVec(v: Vec2): Vec2
multiplyMat(m: Mat2): Mat2
determinant(): number
inverse(): Mat2 | null
transpose(): Mat2
lerp(to: Mat2, t: number): Mat2
```

### Camera2D

负责世界坐标和屏幕坐标转换。

必须提供：

```
worldToScreen(v: Vec2): Vec2
screenToWorld(v: Vec2): Vec2
```

## 需要抽象的场景对象

1. Grid；
2. VectorArrow；
3. UnitSquare；
4. Label；
5. Scene。

## 第二阶段验收标准

1. 第一阶段功能保持不变；
2. React 组件中不再充斥大量数学计算；
3. 坐标转换统一由 Camera2D 处理；
4. 新增一个向量或图形对象时，不需要大量改旧代码；
5. Mat2 的乘法顺序经过测试。

------

# 第三阶段：动画语义系统

## 目标

实现一个简化版动画系统，让项目逐渐具备类似 Manim 的表达方式。

希望未来可以写出类似：

```
scene.play(applyMatrix(grid, A), { duration: 1.5 });
scene.play(fadeIn(label), { duration: 0.5 });
scene.play(applyMatrix(vector, B), { duration: 1.5 });
```

## 核心对象

### Animation

动画接口。

```
interface Animation {
  duration: number;
  start(): void;
  update(t: number): void;
  finish(): void;
}
```

### Timeline

负责调度动画。

初期至少支持：

```
play(animation: Animation): void
update(deltaTime: number): void
pause(): void
resume(): void
reset(): void
```

### Easing

先实现：

```
linear(t: number): number
smoothStep(t: number): number
easeInOutCubic(t: number): number
```

## 第三阶段验收标准

1. 可以通过 Timeline 播放矩阵变换动画；
2. 可以暂停、继续、重置动画；
3. 可以顺序播放两个矩阵变换；
4. 可以演示“先 A 后 B”；
5. 可以显示最终组合矩阵 C = B * A；
6. 动画逻辑不依赖 React 组件内部细节。

# 第四阶段：完整教学模块

## 目标

在底层数学对象和动画系统稳定后，逐步开发完整教学模块。

------

## 模块一：线性变换

功能：

1. 输入矩阵 A；
2. 显示网格变化；
3. 显示 e1、e2 被送到哪里；
4. 显示单位正方形如何变成平行四边形；
5. 显示 det(A)；
6. 说明面积缩放和方向翻转。

核心解释：

```
矩阵的两列就是变换后的基向量。
```

------

## 模块二：矩阵乘积

功能：

1. 输入矩阵 A；
2. 输入矩阵 B；
3. 动画先播放 A；
4. 再播放 B；
5. 显示最终组合矩阵 C = B * A；
6. 支持“分步播放”和“一步播放”的对比。

核心解释：

```
矩阵乘法表示线性变换的组合。
先 A 后 B，对应 B * A。
```

------

## 模块三：基变换

功能：

1. 输入或拖拽新基向量 b1、b2；
2. 显示标准基 e1、e2；
3. 显示新基 b1、b2；
4. 显示新基网格；
5. 如果 b1、b2 线性相关，提示不能构成基。

核心公式：

```
P = [b1 b2]
x = P [x]_B
```

------

## 模块四：坐标变换

功能：

1. 固定一个几何向量 x；
2. 显示它在标准基下的坐标；
3. 显示它在新基 B 下的坐标；
4. 强调向量本身没有变，坐标数字变了。

核心公式：

```
[x]_B = P^{-1} x
```

------

## 模块五：同一线性变换在不同基下的矩阵

功能：

1. 输入标准基下矩阵 A；
2. 输入新基矩阵 P；
3. 显示新基下矩阵：

```
A_B = P^{-1} A P
```

解释路径：

```
[v]_B --P--> [v]_E --A--> [Av]_E --P^{-1}--> [Av]_B
```

------

## 5. 视觉设计目标

最终风格参考 3Blue1Brown：

1. 深色背景；
2. 细网格线；
3. 清晰坐标轴；
4. 高亮基向量；
5. 平滑动画；
6. 简洁公式；
7. 图形和公式同步变化；
8. 避免杂乱 UI。

第一阶段不用追求完美视觉，先保证数学正确和动画流畅。

------

## 6. 最终愿景

最终软件应该像一个“交互式线性代数动画实验室”。

用户可以通过拖拽、输入和播放动画理解：

1. 矩阵如何改变平面；
2. 矩阵乘法为什么是变换组合；
3. 基变换为什么不是改变向量本身；
4. 坐标为什么依赖基；
5. 行列式为什么表示面积缩放；
6. 逆矩阵为什么是“撤销变换”；
7. 特征向量为什么在变换后仍留在原方向上。
