## 当前阶段

当前处于：

```text
第一阶段：Web 最小原型
```

当前目标只有一个：

```
实现一个可以输入 2×2 矩阵并播放二维线性变换动画的最小版本。
```

------

## 第一阶段必须实现

当前阶段需要实现：

1. 创建 Vite + React + TypeScript 项目；
2. 页面采用左右布局；
3. 左侧是控制面板；
4. 右侧是 Canvas 绘图区；
5. 控制面板包含 2×2 矩阵输入框；
6. 默认矩阵为单位矩阵；
7. 控制面板包含“播放变换”按钮；
8. 控制面板包含“重置”按钮；
9. Canvas 显示二维坐标轴；
10. Canvas 显示基础网格；
11. Canvas 显示标准基向量 e1 和 e2；
12. Canvas 显示单位正方形；
13. 点击播放后，执行从 I 到 A 的动画；
14. 动画由 requestAnimationFrame 驱动；
15. 动画插值矩阵为：

```
M(t) = (1 - t)I + tA
```

1. 每一帧用 M(t) 作用到网格点、基向量和单位正方形；
2. 坐标转换必须通过 Camera2D 完成；
3. 非法输入不能导致程序崩溃；
4. 点击重置后恢复到初始状态。

## 第一阶段验收标准

完成后，必须可以手动验证以下情况。

### 测试 1：单位矩阵

输入：

```
[1 0]
[0 1]
```

预期：

```
网格、基向量、单位正方形保持不变。
```

------

### 测试 2：x 方向缩放

输入：

```
[2 0]
[0 1]
```

预期：

```
x 方向拉伸 2 倍。
```

------

### 测试 3：y 方向缩放

输入：

```
[1 0]
[0 2]
```

预期：

```
y 方向拉伸 2 倍。
```

------

### 测试 4：逆时针旋转 90 度

输入：

```
[0 -1]
[1  0]
```

预期：

```
e1 从 (1,0) 变到 (0,1)。
e2 从 (0,1) 变到 (-1,0)。
整体网格逆时针旋转 90 度。
```

------

### 测试 5：水平剪切

输入：

```
[1 1]
[0 1]
```

预期：

```
单位正方形变成平行四边形。
e1 不变。
e2 变成 (1,1)。
```

------

### 测试 6：压缩到直线

输入：

```
[1 0]
[0 0]
```

预期：

```
平面被压缩到 x 轴方向。
单位正方形面积变为 0。
```

------

### 测试 7：方向翻转

输入：

```
[-1 0]
[ 0 1]
```

预期：

```
图形关于 y 轴翻转。
```

------

## 进入第二阶段的条件

只有当第一阶段满足以下条件后，才进入第二阶段：

1. 基础动画可以正常播放；
2. 坐标轴和网格显示正确；
3. 矩阵输入可以正常工作；
4. 重置功能正常；
5. 至少通过 ROADMAP 中列出的 7 个手动测试；
6. 代码没有明显混乱到无法维护；
7. 数学约定没有错误。
